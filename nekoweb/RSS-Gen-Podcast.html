KB's RSS Generator for Podcasts

KB's RSS feed generator for podcasts

To make standard RSS feeds, use this version.

If you are starting from scratch, you will need to make your RSS channel. You only need to do this once for each podcast (not each episode), and when it's done you'll only need to make your items/episodes.

As defined in the specification, a podcast's channel must have the title, description, image, language, category and <itunes:explicit> elements, and at least one item:

Podcast/Channel data Image: Available categories If your podcast has multiple links for funding, please duplicate the <podcast:funding> tag generated changing the attribute(s) If your podcast already has been assigned a GUID, please input it here. Otherwise, it will be generated automatically based on your inputed feed link, as mandated by the specification

As defined in the specification, <item> elements should be inside the <channel> tag, after all its other elements, and their order is not important. However, I recommend ordering items by date, keeping newer ones near the top, just to be consistent.

As defined in the specification, there must be at least one item, and it must contain at least a title, enclosure (the file for the podcast) and guid (basically a link or a unique identifier for the episode). I recommend including all of these (except stated otherwise):

* Subtitles/closed captions/transcription tools I found:

  • vizard.ai: must sign up but subtitle/transcript download seems free
  • turboscribe.ai: 3 30min long file transcripts daily for free
  • youtubetotranscript.com: if you provide a YT link, it gets the transcript in plain text for free
  • veed.io: subtitle/transcript download was free at some point, it doesn't seem like it anymore

Privacy notice: This demo runs entirely within your browser. In other words, no data is sent to me, the code is sent to your browser and runs locally. You could theoretically load the page, cut your network connection and then use it (external feed validation and URL based file validation would fail, though).
